1. The Tech Stack

Frontend: React (Dashboard for uploads, viewing vault, and triggering DMCA actions).

Backend Receptionist: FastAPI (Python).

Message Broker: Redis (Docker).

The Heavy Lifter: Celery Workers (Python).

The Vault: PostgreSQL with pgvector extension (Docker).

The Radar: Chrome Extension (JavaScript/MutationObserver/Bloom Filter) & Python Scrapers (PRAW/Telethon for Danger Zones).

2. The Backend Protection Pipeline (Working State)

User uploads an image via React.

FastAPI saves the file temporarily and immediately pushes a ticket to Redis (.delay()), responding to the user instantly to prevent bottlenecking.

Celery picks up the ticket, calculates the real pHash for the Mesh Network, applies adversarial noise (NOISE_EPSILON = 0.02), injects an invisible watermark (Steganography), and extracts the 128-dimension face encoding array.

Celery saves the User ID, pHash, Watermark ID, and Face Vector into a single row in the unified pgvector database.

Celery deletes all temporary files to prevent server crashes.

3. The Radar Pipeline (The Mesh Network)

The Chrome Extension observes the DOM (e.g., Instagram feed) without sending every image to the server.

It calculates the pHash of visible images locally on the user's machine.

It checks the local hash against a lightweight Bloom Filter of protected hashes.

Only if there is a match does it ping the FastAPI backend with the suspect hash and URL.

4. The Action Pipeline (Manual Trigger)

To avoid legal liability, AMOR does not automatically issue DMCAs.

When the Radar spots a match, the backend emails the user: "AMOR spotted your image at [URL].".

The user logs into the React Dashboard to review. If it is malicious, the user clicks "Generate Takedown," and the backend generates/sends the DMCA.

1. The AMOR Hivemind (The Mesh Network)

You assumed the Chrome extension only looks for that specific user's photos. That is not how we are going to build it.



The Architecture: When User A installs the AMOR extension, it downloads a lightweight, compressed list of everyone’s pHashes.

The Execution: When User A is scrolling through Reddit on their laptop, the extension isn't just looking for User A's face. It is looking for any AMOR user's face.

The Result: If User A stumbles upon a deepfake of User B, the extension silently flags your server, and User B gets the DMCA alert email instantly.

The Power: If you have 10,000 users, you have 10,000 decentralized bots scraping the web for free, 24/7. You don't need to sweep the internet; your users' collective browsing habits are the sweep.



2. The Danger Zone Bots (Free OSINT)

The Architecture: Deepfakes don't usually go viral on mainstream Instagram first. They are born in the "Danger Zones"—specific 4chan boards, dedicated Reddit communities (like r/SFWdeepfakes), and public Telegram channels.

The Execution: You write three simple Python scripts using free wrappers: PRAW (for Reddit), Telethon (for Telegram), and Discord.py.

The Flow: These scripts live on your backend. They don't search the whole internet. They just sit in those 20 specific degenerate groups and listen. The second an image is uploaded to those groups, the bot grabs it, hashes it, and checks your pgvector database.

The Cost: API access to Reddit, Discord, and Telegram bots is completely free.
